## Purpose / context

An agent picking up an artifact sits inside a graph — the work it extends, the decisions it feeds, its lifecycle state, prior related sessions. That graph is authored but latent (frontmatter relation lists, free-text citations, prose), so every session re-derives context by hand (~4 tool calls + 3 reads per task, measured on the 2026-06-05 reference trace) or acts on a stale picture. This spec locks the designed tier of the retrieval feature that collapses that reconstruction to **one trust-labeled, budget-shaped MCP call**.

The closed world is the locked Frame (blob `65b1d05`). This spec maps Frame directions R1–R11 to requirement slots **R-0023–R-0035** at spec precision; nothing outside the Frame's locked directions enters here. The three companion ADRs resolve the Frame's open slots: [P-0014](../src/adrs/P-0014-retrieval-architecture.md) (retrieval topology + the P-0010 D6 method-borrow landing), [P-0015](../src/adrs/P-0015-provenance-envelope-source-roles.md) (the two-axis envelope contract, decision port, freshness/displacement schema), [P-0016](../src/adrs/P-0016-edge-schema.md) (superset edge schema, extraction, traversal contract). Where a requirement below binds a contract detail those ADRs own (a dimension table, a mapping table, DDL), the ADR is the schema authority and the requirement cites it.

## Requirements

RFC-2119 keywords used throughout. `SHALL`/`MUST` = mandatory. `SHALL NOT`/`MUST NOT` = prohibited. `SHOULD` = preferred. `MAY` = optional. Every acceptance criterion is written to be satisfied only by running the protocol it names — none is passable by pasting a block of text — and every instrument an AC demands is itself an in-scope deliverable of this cluster (R-0034).

**Verification instruments.** Each *(Binary-observable: …)* clause names its instrument where the observable is not a black-box protocol run. Three instrument classes are used: **[acceptance test]** — a runnable black-box scenario through an external surface (MCP verbs, process startup, the test harness's host-fn invocation path); **[construction + review audit]** — a code-shape invariant (signature discipline, uniqueness-of-path, diff boundedness) enforced by construction where stated and verified at code review / CI static check, deliberately NOT an acceptance-suite scenario; **[operator SQL surface]** — a direct workspace-scoped Postgres read of the operator/eval-consumer tables (declared in [§ Data Model](#data-model)). ### R-0023 — Retrieval is a host subsystem; three host-served verbs

*Locks Frame R1 + the verb surface of R2. Anchors: P-0002 — retrieval reads across every artifact family; cross-plugin aggregation is a projection concern, not a substrate concern; P-0013 — plugin domain/non-CRUD verbs are deferred past V0, and host placement keeps that trip-wire unfired; P-0012 — the verbs ride the locked `rmcp` handler contract. [Frame R1; QA-1]*

- **R-0023-a (host placement):** The retrieval subsystem — index builder, edge extractor, query path, traversal engine, envelope assembly — SHALL be host code inside the mnemra-core host process. The three retrieval verbs SHALL be registered on the **existing single stdio MCP server** (the `rmcp` `ServerHandler`, per P-0012) as host-served verbs, exactly as builtin verbs are. They SHALL NOT be exports of any plugin; no plugin domain-verb dispatch machinery SHALL be built for them (P-0013's deferred dynamic-resolution path stays unbuilt). *(Binary-observable: the three verbs answer on the stdio server with no plugin loaded beyond the V0 set; the plugin export surface (the fixed typed `content` interface) is unchanged by this cluster.)*
- **R-0023-b (verb surface is closed):** Exactly three retrieval verbs SHALL exist at V0: `get_context_for(artifact_id, budget?)`, `search(query, budget?, filters?)`, `get_artifact_by_citation(ref, budget?)` (shapes in [§ API Contract](#api-contract)). Additional retrieval verbs SHALL NOT ship in this cluster. *(Binary-observable: the MCP tool list contains these three retrieval tools and no other retrieval-family tool.)*
- **R-0023-c (auth/dispatch order):** Every retrieval verb SHALL follow the locked handler contract: authentication check before routing; a single `WorkspaceCtx` construction site; a per-verb capability check before dispatch (P-0012, P-0006). *(Binary-observable, split by instrument: an unauthenticated retrieval request is refused before any retrieval code runs — [acceptance test]; every retrieval host function takes `WorkspaceCtx` as its first parameter, R-0035 — [construction + review audit], with QA-6's zero-cross-workspace-leakage run as the behavioral proxy.)*

### R-0024 — Disclosure-budget mechanics

*Locks Frame R2. Progressive disclosure is load-bearing physics (intake hard constraint): the system shapes context to a budget; "return more tokens" is not an acceptable failure mode. Anchors: intake SC3; P-TrustworthySignal — a response that hides its own reduction lies; P-MinBlastRadius — one counting function, one assembly point. [Frame R2; QA-2]*

- **R-0024-a (budget default + override):** Every retrieval verb SHALL enforce a disclosure budget: **8,000 tokens default**, overridable per call via the `budget` parameter. The default SHALL be a single-sourced configuration value (eval-calibrated later; the *tunability* is the architectural commitment, the number is a default). A per-call override SHALL affect that call only. *(Binary-observable: QA-2 measure 4 — two concurrent calls with different overrides each observe their own effective budget.)*
- **R-0024-b (deliberate reduction ladder; no mid-item truncation):** An over-budget candidate set SHALL be reduced in this fixed order: (1) rerank-order the candidate set; (2) substitute coarser hierarchy nodes for leaf chunks (collapsed-tree resolution shift); (3) drop lowest-ranked whole items. A returned item SHALL be whole at its resolution (leaf or summary node); **mid-item truncation SHALL NOT occur**. *(Binary-observable: QA-2 measures 1–2 — returned payload token-count ≤ effective budget under the single counting function, and no returned item's content is a proper prefix/infix of its stored node content.)*
- **R-0024-c (budget report; omission visible and recoverable):** Every retrieval response SHALL carry a budget report: requested budget, tokens returned, reduction actions taken (from the closed action set in [§ API Contract](#api-contract)), and the **citations of omitted candidates**. Each omitted citation SHALL resolve via `get_artifact_by_citation`. Omission SHALL NOT be silent. Policy-filtered items are categorically different and SHALL NOT appear in the omitted-candidates list (they were never the caller's to see — R-0025-d; a caller-visible marker would be an existence-disclosure oracle). *(Binary-observable: QA-2 measure 3.)*
- **R-0024-d (one counting function; tokenizer pinned behind it):** Exactly one host-side token-counting function SHALL exist; every budget decision (enforcement, report, caps in R-0030-d) SHALL flow through it. The V0 tokenizer behind that function is **the host-resident embedding-model tokenizer** (the HuggingFace `tokenizers` runtime already in-process via `fastembed-rs` for BGE-M3) — no new dependency, deterministic, offline. Budget numbers are counted in that tokenizer's units; the tokenizer is an implementation behind the counting-function seam and MAY be swapped without contract change. Uniqueness SHALL be enforced by construction: the counting function is the retrieval path's only public counting symbol; any counting helper is private to that function's module and reachable only through it, so a second counting implementation cannot compile into the call path. *(Anchors: P-StackDiscipline S1 — the in-stack, already-resident path covers the requirement; P-LockContract — the counting function is the lock, the tokenizer varies behind it. Binary-observable, split by instrument: two call sites counting the same payload return identical counts — [acceptance test], a behavioral proxy; the single-entry-point construction discipline above — [construction + review audit].)*

### R-0025 — Provenance envelope: policy axis (permissions record, enforcement, decision port)

*Locks Frame R3's policy side at spec precision; [P-0015](../src/adrs/P-0015-provenance-envelope-source-roles.md) is the contract authority (dimension tables, validators, port contract). Keystone anchor: P-SecurityLayered — the envelope's policy dimensions are an authorization control on the retrieval path, not metadata; a retrieval path that ignores them leaks by design (intake risk profile (a)). Co-anchors: intake SC2 (implemented per the Frame §11 interpretation note); P-ShiftLeft D2 — validator before field. [Frame R3; QA-4; QA-9]*

- **R-0025-a (two orthogonal axes):** Every returned context item SHALL carry an envelope `{trust, freshness, citation}` (wire shape in [§ API Contract](#api-contract)). The **policy** axis SHALL be a permissions **record** of orthogonal, independently-valued dimensions — never a single enum — consumed as an **enforcement input** at the enforcement points; it SHALL NOT be served as a label, except where the contract requires it (the `dont_use` citation stub's policy reason, R-0025-e). The **trust** axis (R-0026) SHALL yield exactly one value per item; each policy dimension SHALL yield exactly one value, mechanically.
- **R-0025-b (the eight locked mechanics bind):** The policy record SHALL conform to the eight mechanics the Frame locked (carried verbatim in P-0015): (1) record shape — orthogonal dimensions; (2) validator-gated — every dimension a mechanical predicate over stored fields/edges, **no inference anywhere in assignment** (SHALL NOT call a model or apply prose judgment); (3) structural fail-closed — an **undecidable** check on a security dimension resolves to **its** restrictive pole, per-dimension, never whole-item; a settable-but-unenforced security value SHALL NOT exist in the schema; (4) named enforcement points — index-admission gate, model-egress gate (index-time and query-time), serving predicate, citation resolution (the point the Frame's mechanics name "citation stub"; the disposition is per-dimension — stub for `dont_use`, not-found for `visibility`, R-0025-e); a dimension without an enforcement point is not a dimension; (5) operation facet — every dimension declares the operation it governs; the write side is anticipated structurally, not designed now; (6) closed-but-extensible at ADR tier — adding a dimension is a P-0015 amendment, never a config knob; (7) permission-field writes are recorded, attributable, authorized writes; (8) single decision port — all host-side enforcement points consult one boundary `decide(principal, action, resource, context) → allow/deny + reason`. **Undecidable is defined per mechanic 3 as: the validator's stored source field is present but malformed/unreadable, or missing where the schema requires presence.** Undecidable is distinct from **not-set**: not-set is a *defined permissive V0 default value per dimension* (R-0025-c) — the rendered form of the Frame's opt-in-labeling residual (i) / P-0015 PE-11 — and is never undecidable. *(Binary-observable: QA-4 measures 5 and 7; QA-9 measures 1–3.)*
- **R-0025-c (V0 dimension set + enforcement + not-set defaults):** The V0 policy dimensions SHALL be exactly the set P-0015 locks: `dont_use` (index-admission + query predicates + citation resolution: metadata-only stub; undecidable pole: excluded), `model_egress` `allow|deny` (model-egress gate; undecidable pole `deny`), `visibility` `workspace|admin-only|owner-only` (serving predicate + citation resolution: not-found per R-0025-e; undecidable pole `owner-only`), `tenant_share` (fixed `workspace-only` at V0). **Not-set defaults (per dimension, permissive — residual (i)/PE-11 opt-in labeling rendered structurally):** every policy column SHALL carry a permissive DDL default — `dont_use` not-set, `model_egress: allow`, `visibility: workspace`, `tenant_share: workspace-only` — applied at ingest and by the G2/G3 migration to the existing (unlabeled) `0.14.0` corpus, so a stored artifact row always holds a decidable value for every dimension **on the sanctioned write paths** (ingest, the R-0025-f gate, migration), and a truly *absent* policy value cannot occur on a stored row via those paths; a value written by a direct substrate-layer bypass of the gate can be malformed, which mechanic 3 (R-0025-b) resolves to that dimension's restrictive pole at read. An unlabeled artifact is therefore served to workspace callers and egress-eligible at V0 (the maintainer-accepted residual (i); the polarity revisit stays with PE-11). Enforcement SHALL be structural at each dimension's named point(s): a `dont_use` artifact SHALL have **no rows** in any chunk/vector/FTS index table (excluded at admission, not filtered at serve time); `model_egress: deny` content SHALL be indexed **locally in full** (local embed/rerank are egress-free per RC-1) and SHALL NOT appear in any egress payload; `visibility: admin-only` SHALL serve only callers whose `WorkspaceCtx` role is `Admin`; `visibility: owner-only` SHALL serve **no one** at V0 (caller identity is undecidable under the workspace+role context — the conservative enforcement IS the V0 semantics, not a settable-but-unenforced value). *(Binary-observable: QA-4 measures 1–3, 5, 6, 8; the DDL defaults are assertable from the schema and from a migrated-corpus row's stored values.)*
- **R-0025-d (caller-silent exclusion; operator-side audit):** Policy withholding SHALL be caller-silent — no count, no placeholder, no marker in the response — and SHALL record an operator-side audit event per withheld decision in the **`policy_decision_audit`** table ([§ Data Model](#data-model)) — a **stored** record (the incident-response trail, parallel to `egress_events`), additionally *emitted* as operational telemetry per the observability baseline (generation⊥storage, P-0010 D8/E1). This is the observability signal that satisfies P-TrustworthySignal here; a caller-visible marker would be an existence-disclosure oracle. Policy filtering produces the caller's *entitled view*; it is not an omission-that-lies. *(Binary-observable: QA-4 measures 3 and 8 — byte-level response comparison shows no withheld-item traces [acceptance test]; the `policy_decision_audit` row exists with the expected `{workspace_id, artifact_id, action, dimension, decision, reason, principal}` shape [operator SQL surface].)*
- **R-0025-e (citation-resolution dispositions):** `get_artifact_by_citation` on a `dont_use` artifact SHALL return a **metadata-only stub** carrying the policy reason and no content (the curatorial kill switch is allowed to say "don't use this"). Citation resolution of a **visibility-withheld** artifact by an unauthorized caller SHALL return the **same not-found response an unresolvable reference returns** — indistinguishable in shape and content (P-0015 locks not-found over stub: visibility withholds existence itself; a stub is an existence oracle) — plus the operator-side audit event. *(Binary-observable: QA-4 measures 1, 4, 8 — the visibility-withheld resolution response is structurally identical to a genuinely-nonexistent ref's response.)*
- **R-0025-f (V0 write gate on policy/trust-affecting fields):** ALL writes to policy/trust-affecting fields — permission flips, freshness overrides — SHALL ride the existing admin-gated content-mutation path (P-0009), with `read_observer` excluded; **no other write path to these fields SHALL exist at V0**. Every such write SHALL be recorded as an attributable row (actor `token_id`, field, old/new value, timestamp — [§ Data Model](#data-model) `policy_write_audit`). *(Binary-observable: QA-7 measure 3 — an override write with a `read_observer` token is refused; the accepted write produces the audit row.)*
- **R-0025-g (decision port + conformance):** All host-side enforcement points (index-admission gate, model-egress gate, citation resolution) SHALL reach policy decisions only through the single `decide(principal, action, resource, context) → allow/deny + reason` boundary; index-time gates SHALL consult it under the named **system principal** (P-0015: a distinguished principal variant, not a workspace role — batch index builds have no caller). The row-level serving predicate is the deliberate exception: it executes in the database and enforces through SQL fragments derived from the port's attribute vocabulary in **exactly one module** — and it SHALL be applied **in every channel's WHERE clause (FTS, dense) and at every recursion level of the traversal CTE**, mirroring the R-0035-a workspace predicate (the Frame C4 keystone: policy + workspace predicates at the earliest structural point), so a policy-restricted row never enters RRF fusion, rerank, or budget accounting — rank order and the budget report's returned/omitted counts are computed over the caller's entitled set only. The same serving predicate SHALL also bind the **relations edge projection** — the `get_context_for` `relations` bundle, the only caller-facing edge projection at V0: an edge SHALL surface to a caller only if the caller is serve-entitled to **both endpoint artifacts** under the `visibility` serving predicate — for the anchor's own edges the anchor endpoint is entitled by construction (a withheld anchor already resolves not-found, R-0025-e); every other endpoint is checked explicitly, **whichever side of the edge it sits** — an incoming edge's *source* endpoint included, and both ends of a deeper-hop edge (no projection may surface an edge whose restricted endpoint happens to be the source rather than the target). The per-level traversal predicate already prevents traversal *through* a restricted node; the projection filter is the additional guarantee on the **returned edge list**, which the reachable-node set does not imply. A withheld edge is caller-silent per R-0025-d — absent from `relations`, indistinguishable from no-edge, no marker — with the `policy_decision_audit` row recorded (action `serve`, dimension `visibility`, the restricted endpoint's artifact id). An edge to a `dont_use` artifact is NOT withheld — curatorial policy announces itself (its citation resolves to the metadata-only stub, R-0025-e) — and `model_egress` does not affect serving; the filter is exactly the audience-restriction dimension (existence withholding binds every caller-facing projection — nodes, citation resolution, and edges alike; P-0015 PE-4). A **port conformance suite** (fail-closed + attributes-only invariants; the P-0010 D5 two-adapter-test pattern) SHALL exist from V0 and run against any implementation behind the port, and a **differential conformance test** SHALL verify the serving-path fragments and `decide()` return identical allow/deny over a generated resource set covering every dimension × pole × undecidable case (the serving-path equivalence form P-0015 mandates). *(Binary-observable, split by instrument: QA-9 measures 3–4 — the standing differential/conformance suite, which demonstrably fails when a seeded fragment/predicate divergence is introduced [acceptance test]; QA-9 measures 1–2 — the substitution-time audit [construction + review audit]; per-channel predicate **outcome** — a seeded restricted row is absent from `items`/`results` and from `budget_report.omitted`, over both an exact-key and a semantic-match fixture (so either channel would otherwise have surfaced it), and the budget report over a fixture with restricted matches counts only entitled candidates [acceptance test]; the placement-**mechanism** half — "in every channel's WHERE clause and at every recursion level", and its relations-projection extension — [construction + review audit], with the outcome test and QA-4 measure 9 as its behavioral proxies (the R-0023-c/R-0026-a/R-0035-a split pattern); relations endpoint-filter outcome — QA-4 measure 9 [acceptance test].)*

### R-0026 — Provenance envelope: trust axis, freshness, displacement, citation

*Locks Frame R3's trust/freshness side; P-0015 owns the mapping and class tables. Anchors: the sources-with-roles finding (maintainer record, 2026-05-21) with the maintainer's two advances; P-PreserveDecisionSpace — displacement semantics (re-eval, never auto-invalidation); P-AgentPrimarySource — ID+name pairing, block addressability. [Frame R3; QA-7]*

- **R-0026-a (trust values + precedence, mechanical only):** The trust axis SHALL yield exactly one of `outdated` / `authoritative` / `background` per item, assigned by first-match precedence `outdated` → `authoritative` → `background`. Every predicate SHALL be mechanical over stored fields and edges; **no LLM call and no prose judgment SHALL appear anywhere in trust assignment**. `authoritative` SHALL fire iff the artifact kind's lifecycle field is in that kind's closed authoritative-state set (the per-kind mapping table P-0015 locks) and `outdated` did not fire; `background` is the total-function default. *(Binary-observable, split by instrument: a fixture artifact in each lifecycle state maps per the P-0015 table, and trust assignment over the full fixture corpus yields identical values in the zero-egress configuration with all four generative placements OFF — the QA-5 harness technique reused as the behavioral proxy for inference-free assignment [acceptance test]; the no-inference-call-in-the-assignment-path claim itself — [construction + review audit].)*
- **R-0026-b (`outdated` predicate + origin weighting):** `outdated` SHALL fire when (a) an incoming `supersedes` edge with origin `declared` or `system` exists, OR (b) freshness state is stale (version-handle diff shows `moved`, or the decay-class TTL expired without re-validation) AND no recorded freshness override exists. An `origin = extracted` supersedes edge SHALL NOT enter the trust predicate at V0 — it is recorded, traversable, surfaced in the relations bundle (to callers serve-entitled to both endpoints — the R-0025-g endpoint filter), and surfaced as an operator-side curatorial signal for confirmation (conversion to `declared` via the R-0025-f write gate), but it SHALL NOT flip trust automatically (the extraction path is an authority-laundering vector — Frame §5 worked case). **The curatorial signal's home is the stored edge row itself**: unconfirmed extracted supersedes edges are enumerable via the operator SQL surface (`edges` where `edge_type = supersedes AND origin = extracted`); no separate event table is needed — the edge is the signal, and the count of new extracted supersedes edges per index run is recorded in the R-0034-d build record. *(Binary-observable: a fixture with only an extracted supersedes edge retains its prior trust value [acceptance test] and the edge row is enumerable as above [operator SQL surface]; the same edge re-written as `declared` through the write gate flips the target to `outdated` [acceptance test].)*
- **R-0026-c (freshness: handle-diff primary, TTL fallback, override structural):** Where an item cites a source with a version handle (closed handle-kind set per P-0015: `git-sha`, `semver`, `model-id`, `doc-version`), the envelope SHALL store the handle-at-index and report `current | moved | unknown` by diffing against the source's live handle; an unresolvable live handle reports `unknown`, never a fabricated state. Handle-less sources fall back to the decay-class TTL (the volatility-matched class table P-0015 locks); the TTL bounds ignorance, it does not measure staleness. A freshness override SHALL be a recorded row `{by, reason, date}` accepted only through the R-0025-f write gate — **never a config toggle**; content-based staleness detection SHALL NOT be used (rejected as circular). An override SHALL restore the prior trust value AND itself be visible in the envelope. *(Binary-observable: QA-7 measures 1 and 3.)*
- **R-0026-d (displacement ≠ staleness):** A named **displacement-event registry** SHALL exist with the **closed V0 event-kind set P-0015 PE-9 locks** — `canon-superseded` (system-fired: a `declared`/`system` supersedes edge lands on a canon-kind artifact), `dependency-major-change` (system-fired: a `semver` handle diff crosses a major version), `operator-declared` (operator-fired through the R-0025-f write gate, naming axis + description) — extensible only by P-0015 amendment ([§ Data Model](#data-model) `displacement_events`); a fired event SHALL set a `re-eval pending (axis)` flag surfaced in the envelope of decisions whose recorded axes intersect the event. Single-axis displacement SHALL trigger re-evaluation only; it SHALL NOT flip the trust value (multi-axis decisions survive a single-axis shift). Decision-kind artifacts SHALL record their decision axes as substrate data. Honest limit, carried verbatim from the Frame: trip-wires only cover enumerable displacement events; truly unforeseen shifts fall back entirely to the volatility TTL. *(Binary-observable: QA-7 measure 4 — constructible with the closed kind set above; an event of an out-of-set kind is rejected at write.)*
- **R-0026-e (stable citation on every item):** Every envelope item SHALL carry a stable citation — artifact ID (content-addressed, per the V0 substrate) plus a block anchor where the item is a chunk — resolvable by `get_artifact_by_citation`. *(Binary-observable: QA-1 measure 3; QA-2 measure 3 — every returned and every omitted citation resolves.)*

### R-0027 — Edge substrate: one superset table, discriminating provenance semantics

*Locks Frame R4; [P-0016](../src/adrs/P-0016-edge-schema.md) is the schema authority (DDL, uniqueness, classing rule, extraction contract). Anchors: Stage 2a direction 1 (decomposer-ratified); P-MinBlastRadius — one vocabulary, one traversal, one extraction path; P-0010 D4 — recursive CTEs serve the shallow edge model; intake SC5. [Frame R4; QA-1; QA-8]*

- **R-0027-a (superset closed vocabulary):** One edge table (extending the `0.8.0` substrate) SHALL carry the closed superset vocabulary `{parent, blocks, depends-on, dispatched-by, extends, feeds, cites, supersedes}`; `supersedes` unifies the two prior sets. Edge types outside this enum SHALL be rejected at write *(Binary-observable: an out-of-enum write is refused [acceptance test])*. There SHALL be exactly **one traversal code path and one extraction code path** over this table *(a code-shape invariant — [construction + review audit], not an acceptance-suite scenario)*.
- **R-0027-b (discriminating columns):** Every edge row SHALL carry `edge_class` ∈ `{work-graph, citation}` and `origin` ∈ `{declared, extracted, system}` (closed enums). `supersedes` SHALL be classed per its writer: `work-graph` iff created by the work-graph system-of-record transactional path (entity lifecycle operations), `citation` otherwise (P-0016's classing rule). Work-graph edges SHALL be created transactionally with their entities; citation edges SHALL be re-derivable.
- **R-0027-c (extraction: idempotent, provenance-carrying, measured):** The edge extractor SHALL derive citation edges from frontmatter relation lists (`origin = declared`) and free-text citations (`origin = extracted`, carrying a source-span provenance pointer back to the text it was read from). Re-extraction SHALL be idempotent (P-0016's uniqueness + origin-authority upgrade semantics: `system` > `declared` > `extracted`; an extractor write SHALL NOT downgrade or duplicate a higher-authority row). **Extraction coverage over the migrated corpus SHALL be measured and reported per index run** — per-source counts of frontmatter relations and free-text citations that resolved to edges vs failed to resolve — never assumed. *(Binary-observable: intake SC5 — running the extractor twice yields identical edge rows; the coverage record exists per run with nonzero resolved/unresolved counts against the real corpus.)*
- **R-0027-d (traversal contract):** Graph traversal SHALL be a recursive CTE over the edge table (P-0010 D4 — no graph engine), carrying the workspace predicate **at every recursion level**, bounded by a per-call depth (default and max per P-0016), and instrumented per R-0034-c. *(Binary-observable: QA-6 measure 2; QA-8.)*

### R-0028 — Tree-build seam: authored-first as default, under a named eval

*Locks Frame R5. Anchors: P-LockContract ⇄ P-PreserveDecisionSpace when-to-lock discriminator — the seam is intrinsic and locks; the winner is separable and preserved (the canon-free cross-check refuted the stronger authored-beats-clustered claim); P-Defer/DF1. [Frame R5; QA-1; QA-5]*

- **R-0028-a (the seam):** Tree construction SHALL sit behind a `TreeBuilder` seam — one contract producing one node shape (leaf chunks → parent-linked summary nodes at section and document grain). Two strategies are designed against it: `authored` (markdown-header hierarchy populates the tree directly) and `clustered` (embed-cluster-summarize, reimplemented — never vendored). **V0 SHALL build the `authored` strategy only**; the `clustered` strategy SHALL remain a designed-but-unbuilt slot behind the same seam. Switching or adding a strategy SHALL NOT change the node shape, the index schema, or any query-path code (the bounded-change property). *(Binary-observable: the node tables carry no strategy-specific columns; the query path reads nodes with no reference to the producing strategy.)*
- **R-0028-b (per-shape granularity):** High-structure shapes (per the P-0014 chunking policy table: decision records, briefs, specs, skills and kin) SHALL be tree-indexed; low-structure shapes (daily logs, code, structured rows) SHALL be indexed at their natural single grain. The policy table is internal (R-0033).
- **R-0028-c (the hierarchy-source eval is named, not built):** Authored-first ships as an **eval hypothesis, not a validated lock**. The spec names the signals (built by R-0034): (a) per-strategy retrieval-failure rate / nDCG on the golden query set (the same harness instrument P-0010 D3's trip-wire uses — one harness, two consumers), and (b) the **hierarchy-coverage measure** — the fraction of corpus tokens under authored hierarchy at section grain or better, reported per shape at every index run (backfillable from the existing corpus on day one, IB1). The eval that *reads* them is a committed-tier plan deliverable (intake non-goal 3/4 firing mechanism), NOT built here. If coverage is low or the clustered arm wins, the pre-designed `clustered` strategy is built behind the seam — a bounded change. *(Binary-observable: the hierarchy-coverage measure exists in every index-build record with per-shape values computable against the real corpus.)*

### R-0029 — Hybrid search: FTS + dense fused by RRF; local rerank

*Locks Frame R6; the four P-0010 D6 method-borrows land in P-0014 (this cluster is D6's named firing event). Anchors: intake SC4; P-0010 D2/D3/D6; the locked retrieval research (2026-05-27, r3) + canon-free cross-check corroboration. [Frame R6; QA-3]*

- **R-0029-a (two channels + RRF + rerank):** `search` SHALL run two channels at V0 — lexical (Postgres native `tsvector`/`ts_rank`) and dense (pgvector HNSW over **all** tree levels, collapsed-tree style: leaf and summary nodes in one KNN) — fused by Reciprocal Rank Fusion as application-side SQL, followed by local cross-encoder rerank (host `rerank`) into budget. *(Binary-observable: QA-1/QA-3 fixtures; the run record (R-0034-a) shows both channels fired and the fusion + rerank stages executed.)*
- **R-0029-b (exact-key retrieval survives fusion):** An exact-key query (a decision ID, a file path, a function name) SHALL resolve via the lexical channel and SHALL rank in the returned set after fusion + rerank — with HyDE ON and with HyDE OFF (the rewrite is a fused channel, never a gating dependency). *(Binary-observable: QA-3 measures 1–2.)*
- **R-0029-c (keyword leg is deliberately not BM25 at V0):** The keyword leg SHALL be native `ts_rank`; `pg_textsearch` SHALL NOT be adopted at V0. The fidelity trip-wire is P-0010 D3's golden-query `ts_rank`-vs-BM25 regression instrument (the harness is the committed-tier eval deliverable; the run records feed it). Additional fusion channels (fact-key, raw-message, ColBERT late-interaction) SHALL NOT ship at V0; each is added only when retrieval-failure analysis in the run records attributes misses to a class no existing channel can serve.
- **R-0029-d (FTS language config single-sourced):** The FTS language configuration SHALL be a single-sourced per-corpus parameter (default `english`), read wherever a `tsvector`/`tsquery` is built. Inline `to_tsvector('english', …)` literals SHALL NOT appear in schema, queries, or index definitions. *(Binary-observable: a repo-wide scan of migrations and query code finds zero inline language literals on the retrieval surfaces; changing the parameter changes every build site.)*
- **R-0029-e (supersession semantics in retrieval; demote vs exclude):** The mechanical condition separating the two dispositions is the **origin-weighted incoming `supersedes` edge** — no separate schema field. **Hard-superseded** SHALL mean: the artifact has an incoming `supersedes` edge of origin `declared` or `system` (the "`superseded-by` forward pointer" is exactly this edge read from the superseded artifact's side — the directional view, not a second mechanism; P-0016 ES-2). Every hard-superseded artifact's nodes SHALL be excluded from default retrieval and reachable by point-in-time queries (the keyed-supersession borrow, keyed here on the explicit edge — P-0014 RA-10 borrow 3). It follows that R-0026-b **predicate-(a) `outdated` items are always excluded, never served-and-demoted**; the served + labeled + demoted disposition applies exactly to **freshness-stale predicate-(b)** items (no superseding edge). An `origin = extracted` supersedes edge triggers neither disposition (R-0026-b). The two dispositions are distinct and SHALL NOT be conflated. *(Binary-observable: fixtures for each case — the hard-superseded node is absent from default results and present in a point-in-time query; the stale item is present, labeled, and ranked below an equally-matching `authoritative` item; an extracted-only-supersedes fixture is served undemoted.)*

### R-0030 — Generative placements: individually degradable; egress boundary; zero-egress configuration

*Locks Frame R7 + the §5 egress design. Anchors: RC-3 + the maintainer's "all of the above for V0, narrow later" ruling (2026-05-27); Stage 2a direction 2 (decomposer-ratified); P-SecurityLayered — the generative placements open a design-time data-egress layer, stated per placement and gated policy-aware; P-TrustworthySignal — degraded-mode honesty is contractual. [Frame R7, §5; QA-4; QA-5]*

- **R-0030-a (four placements, four independent switches):** All four generative placements SHALL ship as specified surfaces — 4.1 chunk-context (index-time), 4.2 HyDE query rewrite (query-time), 4.3 synthesis (query-time, optional), 4.4 tag generation (index-time) — each behind an **independent config switch**. No narrowing decision is made in this artifact (RC-3); narrowing is eval-time, reading the R-0034 signals.
- **R-0030-b (degraded modes specified; absent ≠ empty ≠ policy-suppressed):** Each placement OFF SHALL degrade exactly per the matrix in [§ API Contract](#api-contract): 4.1 → chunks embed without situating context, rows marked `context_state: absent` so a later enable re-enriches selectively; 4.2 → query embedded directly (direct-vector + FTS only); 4.3 → raw reranked chunks returned; 4.4 → rows carry `tags_state: absent`, and a tag-filtered query returns a **structured degraded-mode notice**, never silently-empty results. Each placement's degraded contract holds **independently of the other three placements' state** (the zero-egress configuration is the all-OFF composition, not the contract's precondition). In every schema this cluster adds, `absent` SHALL be distinguishable from `empty`. Additionally, a row whose artifact's `model_egress: deny` bars 4.1/4.4 egress **while the placement is ON** SHALL be marked **`policy-suppressed`** — distinct from `absent`: the re-enrichment/backfill set is driven by `absent` markers only, so a permanently egress-denied artifact is never re-attempted; a `policy-suppressed` row re-enters the enrichment set only when the artifact's egress label changes through the R-0025-f write gate (a recorded, mechanically detectable event). The capability surface SHALL report per-placement on/off state. *(Binary-observable: QA-5 measures 3–4; a deny-marked fixture row under placement-ON shows `policy-suppressed`, and a subsequent index run's egress log shows zero attempts for it.)*
- **R-0030-c (zero-egress is a supported V0 configuration):** With all four generative placements OFF and local encoders on, indexing (chunking, authored-tree build, local embed, FTS, edge extraction) and all three verbs (hybrid search + rerank + envelope + traversal) SHALL be fully functional with **zero outbound network connections at runtime** (RC-1: BGE-M3 + BGE-reranker host-side). The claim is scoped to **operation**: the encoder model artifacts are **provisioned at deploy time, never fetched at runtime** — in egress-permitted environments a deploy/first-provision step MAY fetch them into the local model cache (the TB-5 `DF-model-fetch` flow, pin+verified per R-0031-d); in a zero-egress environment they are placed **out-of-band** (pre-provisioned cache or baked into the deployment artifact). A host starting in the zero-egress configuration with a required model artifact absent SHALL fail the model load closed per R-0031-d — it SHALL NOT attempt a network fetch. This is the sovereignty reconciliation: "data never leaving that boundary" is satisfiable at V0 by configuration plus out-of-band provisioning. *(Binary-observable: QA-5 — with model artifacts pre-provisioned (the stated fixture precondition), the reference-trace fixture passes under a test harness that denies all non-loopback egress; the same harness with an artifact removed observes a structured load failure and zero fetch attempts.)*
- **R-0030-d (policy-gated, bounded egress in the default configuration):** In the egress-on configuration, every egress SHALL pass the model-egress gate: no chunk of a `model_egress: deny` or `dont_use` artifact SHALL ever be sent to the external LLM API (index-time or query-time). Egress SHALL be bounded: index-time runs SHALL enforce a per-run document-token budget (default **2,000,000 tokens**, counted by the R-0024-d function) and a concurrency bound (default **4** concurrent egress requests, **scoped per-workspace** — consistent with R-0035's tenancy discipline, so one workspace's index run cannot starve another's egress slots). **No process-wide egress-concurrency ceiling exists at V0**: the token budget bounds each run's *volume*, not simultaneous connections, so N concurrently-indexing workspaces can present up to 4·N concurrent provider connections — accepted plainly at the stated V0 scale (single-maintainer dogfood + small adopter deployments, tens of index runs); the `egress_events` records (R-0034-b — per-event `provider` + `occurred_at`) are the instrument that would show aggregate connection pressure, and a process-wide ceiling is a single added config key if it does. Query-time calls SHALL enforce request-size caps (HyDE input cap default **2,000 tokens** — an over-cap query skips the rewrite and falls back to direct embed, recorded; synthesis payload cap default **16,000 tokens** — items beyond the cap are excluded whole, never mid-chunk truncated). All defaults are config-tunable; the caps' existence and enforcement are the lock, the numbers are eval-calibrated defaults. An index run that exhausts its budget SHALL defer the remaining documents to the next run and record the deferral **per document** — the deferred document ids, not only a count, land in the R-0034-d build record. *(Binary-observable: QA-4 measure 2 — index-time and query-time egress logs show zero payloads for the deny-marked artifact; a fixture run exceeding a cap shows the enforcement action in its run record.)*
- **R-0030-e (synthesis is optional-in-ABI and marked untrusted; response-level digest):** The `summarize` primitive SHALL be optional-in-ABI: its availability is a capability the caller can probe; absence is a structured state, not an error surprise. The 4.3 digest is **a single response-level object, not per-item content substitution** (Frame §5: synthesis "folds retrieved content into a digest"): when 4.3 is ON and the response payload contains at least one egress-eligible item, the `search`/`get_context_for` response SHALL carry a top-level **`synthesis: { digest, untrusted: true }`** field ([§ API Contract](#api-contract)); returned `items`/`results` remain original corpus text. The `untrusted: true` marker SHALL live on that response-level object — the digest is **model-generated** content the consuming agent must not treat as instructions (Frame §5 residual (iv) control (b)). `model_egress: deny` items SHALL never be included in a synthesis payload; a response containing such items skips synthesis for those items; a response with no egress-eligible items carries no `synthesis` field. The digest is **inside the caller's disclosure budget** (R-0024): its tokens are counted by the R-0024-d function and included in `budget_report.returned_tokens`, and the response's total counted payload — items plus digest — SHALL NOT exceed the effective budget; when the item payload leaves no headroom for a digest, the `synthesis` field is omitted whole (absence is already the structured state — no second cap mechanism, no separately-uncounted served payload). *(Binary-observable: with 4.3 ON the fixture response carries `synthesis.digest` with `untrusted: true`, no item's `content` is model-generated, and `budget_report.returned_tokens` includes the digest's counted tokens with the total counted payload ≤ the effective budget; with 4.3 OFF the field is absent and raw reranked chunks return.)*
- **R-0030-f (chunk-context output is bounded, embed-only, never served):** 4.1 chunk-context output — model-generated text derived from untrusted corpus content with no closed vocabulary to validate against — SHALL be controlled structurally (the P-0014 residual-(iv) control): stored in a dedicated column marked model-generated; length-capped (default **200 tokens**, over-cap output rejected and the row marked context-absent); used **only** as dense-embedding input for its chunk; **excluded from the FTS index and from every served payload** (served *items* are original corpus text only; the sole served model-generated content anywhere in a response is the explicitly-marked response-level 4.3 digest, R-0030-e). Its influence is thereby bounded to dense-ranking shift — it can never inject served content or instructions. *(Binary-observable: no served response contains chunk-context text; the FTS column's source expression excludes the chunk-context column; an over-cap fixture row is context-absent.)*

### R-0031 — Host-fn ABI extension: five inference primitives

*Locks Frame R8 + TB-5. Anchors: RC-1 — local non-generative inference permitted host-side, generative never hosted; the locked retrieval research §3/§5.6 + OQ5 — lock the ABI shape now to avoid breaking it at plugin activation; P-0002/P-0012/P-0013 — IO-free cores, host-mediated IO; P-StackDiscipline S1 — `fastembed-rs` is the in-stack path, all Green tier. [Frame R8; QA-5]*

- **R-0031-a (five primitives, host-implemented):** The host-fn surface SHALL extend with five inference primitives — `embed`, `rerank`, `rewrite-query`, `summarize` (optional-in-ABI), `classify` (signatures in [§ API Contract](#api-contract)) — implemented host-side: local ONNX via `fastembed-rs` for `embed`/`rerank` (BGE-M3, BGE-reranker-v2-m3); the configured external API for the generative three at V0. Inference SHALL NOT run inside a plugin (ONNX is a native dependency the WASM sandbox cannot host; one model serves all consumers).
- **R-0031-b (shape locks now; import direction only):** The primitive shapes SHALL be declared in the WIT world now, even though the V0 consumer is the host's own retrieval subsystem, so plugin-facing activation at V0.1+ is additive, not an ABI break. - **R-0031-c (structural unavailability; external-call bounds):** An inference host-fn that is unavailable (placement disabled, model not loaded, provider unreachable) SHALL return a structured error or a structured absent-capability state; it SHALL NOT no-op, return empty-success, or silently degrade. Every external call SHALL be time-bounded and single-attempt: per-class timeout defaults — `rewrite-query` **10 s** (it blocks a query-time verb), `summarize` **30 s**, `classify` **30 s**, deploy-time model fetch **300 s** — all config-tunable ([§ Config surface](#config-surface)); the posture is **one attempt, no automatic retry, fail structured** — a timed-out or failed call surfaces the structured provider-failure variant and the placement's degraded fallback applies (4.2 → direct embed, recorded; 4.3 → raw chunks; 4.4/4.1 → the affected document is deferred to the next index run, riding R-0030-d's deferral mechanics — no in-run retry). A persistently-failing document has **no repeat-deferral cap and no dead-letter path at V0**: a document whose call deterministically fails is re-attempted and re-deferred each run (mechanism ahead of evidence — P-Defer, the same posture as the circuit breaker below); the deferral record is per-document (R-0030-d / R-0034-d: deferred document **ids**, not only a count), so a stuck document is mechanically distinguishable from N once-deferred documents — the same artifact id recurring across consecutive build records' deferred sets is the firing signal for the cap/dead-letter design [operator SQL surface]. No circuit breaker ships at V0 (mechanism ahead of evidence — P-Defer); the run/build records' failure counts are the instrument that would motivate one. *(Binary-observable: calling `summarize` with 4.3 disabled returns the structured absent state, not an empty digest — the five primitives have no external caller surface at V0, so host-fn-level ACs like this one are exercised by the test harness invoking the host-fn layer in-process [acceptance test]; a fixture provider stub that hangs past the timeout yields the structured failure within the bound, exactly one attempt in the provider log, and the recorded fallback.)*
- **R-0031-d (model pin + integrity verification — TB-5):** The model-config schema SHALL pin each local model's **revision** and verify artifact **integrity** (a fixed digest from a trusted source) before load; unpinned or `latest` model references SHALL NOT be accepted — a swapped or tampered encoder silently changes every embedding and rerank score. A failed integrity check — and an absent artifact in a configuration that forbids fetching (R-0030-c) — SHALL fail the model load closed with a structured error; while a required encoder model is unloaded, the `/health` endpoint's `overall` field SHALL report `"degraded"` (the R-0007-h precedent for host-level operational failures) and the retrieval verbs SHALL return the R-0031-c structured unavailable state. *(Binary-observable: a config with an unpinned model ref is rejected at startup; a fixture artifact with a wrong digest fails load with the distinct error; with an encoder unloaded, `/health` reports `"degraded"` and a retrieval verb returns the structured unavailable error.)*

- **Per-user identity machinery** — owner-columns-as-identity, caller identity in `WorkspaceCtx`, real owner==caller checks. - **Write-side policy dimensions** (operation facet write/label/supersede). Anticipated structurally (R-0025-b mechanic 5), not designed. *(Self-announcing: the first surface granting a non-owner actor write/label capability — per-user identity, `tenant_share` sharing, or plugin-mediated writes. Until then the write surface is the R-0025-f admin-gated flip.)*
- **Timed embargo / auto-expiring permission values.** Recorded as a P-0015 design option, unbuilt. *(Self-announcing: a real embargo case is asked for.)*
- **The clustered tree-builder.** Designed slot behind the `TreeBuilder` seam, unbuilt (R-0028). *(Instrumented: the hierarchy-source eval read — R-0028-c signals.)*
- **Adopter-facing index-granularity flag** (R-0033). *(Self-announcing adopter ask, or the 3.0× storage-overhead instrument.)*
- **Local generative models.** V0.1+ arrivals through the same host-fn seam. *(Instrumented: the R-0034-b egress-volume counter ≥ ~14M doc-tokens/month, ~$50/mo rewriter spend, a p99 latency bound, or a sovereignty requirement — self-announcing.)*

- **Accessibility work faked into this MCP-verb cluster.** Accessibility is routed to the product brief as a standing product requirement binding on UI/docs surfaces (amended with this change); this cluster has no human-facing UI surface.

## Data Model

Field tables, not SQL; exact DDL, uniqueness constraints, and index choices are governed by [P-0016](../src/adrs/P-0016-edge-schema.md) (edges/traversal) and the implementation tier (other tables), within these binding shapes. **Every table adds `workspace_id NOT NULL` (indexed, explicitly passed) per R-0035-a; content-bearing tables add `owner`/`created_by` per R-0035-c; all are plain timestamped Postgres tables (P-0010 D8 — no TimescaleDB).**

**Operator read surface (declared).** The instrumentation and audit entities — `retrieval_runs`, `traversal_log`, `egress_events`, `index_builds`, `policy_write_audit`, `policy_decision_audit` — are **operator/eval-consumer tables with no caller-facing verb**: their declared read surface at V0 is **direct workspace-scoped SQL against the tables** (the *operator SQL surface* — used by the operator, the eval/acceptance harness, and the D4/D5 trip-wire reads). This is deliberate: the three-verb surface is closed (R-0023-b), and these records are not the caller's to see. Every "record exists / is queryable" AC in this spec names this surface as its instrument. The four caller-relevant entities keep their existing external observables (`citation_handles` → envelope `freshness`; `edges` → `relations`, endpoint-filtered per R-0025-g; `displacement_events` → `re_eval_pending`; `retrieval_nodes` → served items). **V0 scoping caveat (note, not a new control):** this surface is **credential-gated, not structurally workspace-enforced** — V0 enforcement is application-layer on the verb path only (P-0006), so direct SQL carries no structural workspace filter until RLS activates at V0.1+ (R-0035-b; the P-0009 preconditions these tables carry); "workspace-scoped SQL" names the operator's query discipline, not an enforced property. The surface requires Postgres credentials — the single maintainer/operator at V0 — and no caller-facing verb reaches it; sensitive contents (`egress_events.payload_audit`, `policy_decision_audit`) sit behind that credential gate until RLS activation.

## API Contract

### MCP verbs (host-served, stdio; R-0023)

Payload field names below are binding at contract level; exact JSON serialization is implementation-tier. Every response carries `budget_report` and `capabilities` (per-placement state).

#### `get_context_for(artifact_id, budget?)` — the context bundle (first-class verb)

- **Request:** `{ artifact_id: string, budget?: int, depth?: int }` (depth default/max per P-0016).
- **Response:** `{ artifact: { id, kind, lifecycle_state }, relations: [ { edge_type, edge_class, origin, direction, target_citation } ], items: [ { content, level, envelope } ], synthesis?: { digest, untrusted: true }, budget_report, capabilities }` — typed relations, lifecycle state, linked-artifact summaries, full envelope per item; `synthesis` present only when 4.3 is ON and the payload had egress-eligible items (R-0030-e). `relations` lists only edges whose **both endpoints** pass the caller's serving predicate — an edge to a `visibility`-withheld neighbor is absent, indistinguishable from no-edge (R-0025-g); an edge to a `dont_use` artifact surfaces (its `target_citation` resolves to the metadata-only stub, R-0025-e).
- **Errors:** auth failure (before routing); unknown artifact → not-found; `dont_use` target → metadata-only stub; visibility-withheld target → not-found-indistinguishable (R-0025-e).

#### `search(query, budget?, filters?)` — first-pass verb

- **Request:** `{ query: string, budget?: int, filters?: { tags?: [string], kind?: [string], point_in_time?: timestamp } }`.
- **Response:** summary-grain results — titles + summaries + citations, not full bodies (full bodies reachable by citation expansion): `{ results: [ { title, summary, envelope } ], synthesis?: { digest, untrusted: true }, budget_report, capabilities, degraded_notice? }`. `synthesis` per R-0030-e (response-level, 4.3-ON only). A tag filter against a tag-gen-disabled corpus returns `degraded_notice`, never silently-empty results (R-0030-b).

#### `get_artifact_by_citation(ref, budget?)` — citation expansion

- **Request:** `{ ref: string, budget?: int }` (artifact ID + optional block anchor).
- **Response:** the artifact/block content with envelope; the second-pass verb behind `search`'s summary grain and `budget_report`'s omitted citations.
- **Errors/dispositions:** unresolvable ref → not-found; `dont_use` → metadata-only stub `{ citation, policy_reason }`, no content; visibility-withheld → not-found-indistinguishable (R-0025-e).

#### Envelope wire shape (per item; R-0025/R-0026)

`{ trust: "authoritative" | "outdated" | "background", freshness: { mode: "version-handle" | "decay-class", handle?, state: "current" | "moved" | "unknown", ttl_class?, override?: { by, reason, date } }, citation: { artifact_id, block_anchor? }, re_eval_pending?: [axis] }` — the policy record is an enforcement input and does not appear on the wire except the `dont_use` stub's `policy_reason`. The synthesis untrusted marker is NOT a per-item envelope field — it lives on the response-level `synthesis` object (R-0030-e).

#### `capabilities` and `degraded_notice` shapes (R-0030-b)

`capabilities`: one boolean per placement id — `{ "4.1": bool, "4.2": bool, "4.3": bool, "4.4": bool }` (binding key set; exact serialization implementation-tier). `degraded_notice`: `{ placement, detail }` — the disabled placement id plus a structured detail naming the disabled capability.

#### Budget report shape (R-0024-c)

`{ requested: int, returned_tokens: int, actions: [ "rerank-ordered" | "coarsened" | "dropped" ], omitted: [ citation ] }` — closed action vocabulary matching the reduction ladder. `returned_tokens` includes the `synthesis.digest` tokens when the field is present — the digest is inside the disclosure budget (R-0030-e).

### Host-fn ABI — five inference primitives (import direction; R-0031)

WIT-shape signatures (contract level; exact WIT text is implementation-tier within these shapes — parameter/result shapes are binding):

| Primitive | Signature shape | V0 implementation | Availability |
|---|---|---|---|
| `embed` | `embed(text: string) -> result<vector<f32>, inference-error>` | local ONNX, BGE-M3 (`fastembed-rs`) | always-on (RC-1) |
| `rerank` | `rerank(query: string, candidates: list<string>) -> result<list<scored-index>, inference-error>` | local ONNX, BGE-reranker-v2-m3 | always-on (RC-1) |

### Degraded-mode matrix (R-0030-b; per-placement OFF behavior)

| Placement | Egresses (when ON, default config) | Degraded mode when OFF |
|---|---|---|
| 4.1 chunk-context (index-time) | Document content — policy-gated; deny-marked chunks skipped and marked `context_state: policy-suppressed` (excluded from backfill — R-0030-b) | Chunks embed without situating context; rows `context_state: absent`; later enable re-enriches selectively (backfillable) |
| 4.2 HyDE rewrite (query-time) | Query text (the config switch is the control — stated honestly) | Query embedded directly; direct-vector + FTS channels only; exact-key retrieval unaffected |
| 4.3 synthesis (query-time) | Retrieved content, top-k post-rerank; deny-marked items never included | Raw reranked chunks returned (the expected steady default for strong consumers); no capability lost |
| 4.4 tag-gen (index-time) | Chunk content — same gating as 4.1 (deny-marked rows `tags_state: policy-suppressed`) | `tags_state: absent` (≠ empty); tag-filtered queries return the structured degraded notice |

Zero-egress configuration = all four OFF + local encoders on (R-0030-c, QA-5).

### Config surface (C8)

The complete config surface this cluster adds: per-placement enable/disable (four independent switches) · disclosure-budget default (8,000) · egress caps (index-run token budget 2,000,000; per-workspace concurrency 4; HyDE input cap 2,000; synthesis payload cap 16,000) · external-call timeouts (rewrite 10 s; summarize 30 s; classify 30 s; model fetch 300 s — R-0031-c) · traversal depth default/max (2 / 8 — P-0016 ES-5) · traversal latency bound (500 ms) · storage-overhead threshold (3.0× — R-0033-b) · instrumentation-storage threshold (5 GB — R-0034-d) · decay-class TTLs (30/90/180/365 d — P-0015 PE-8, amendable per class) · chunk-context length cap (200 tokens — R-0030-f) · FTS language (per-corpus, default `english`, single-sourced) · model config (model id + **pinned revision** + **integrity digest** per R-0031-d) · provider config (rides the `0.1.0` LLM-API-key surface). Config values SHALL NOT create policy semantics (R-0025-b mechanic 6: dimensions are ADR-tier, never config knobs); the per-shape granularity policy table is deliberately NOT config (R-0033-a).

**Storage + validation:** these values ride the same deploy-time file-based configuration surface the `0.1.0` provider config uses (R-0014-d's home) — not database state. Startup validation is fail-closed on invalidity, defaulting on absence: a **missing** value takes its spec-named default above; a **present-but-invalid** value (unparseable; zero/negative where a positive bound is required) SHALL be rejected at startup with a structured error naming the key (the R-0031-d reject-at-startup precedent).

### Decision port (internal contract; R-0025-g)

`decide(principal, action, resource, context) → allow/deny + reason` — principals: caller (`WorkspaceCtx` workspace + role) or the named system principal (index-time); actions: `serve | egress | resolve-citation | admit-to-index`; resources: content items with their policy-dimension attributes; attributes-only (the port consults stored fields, never content). V0 implementation: hardcoded validator predicates behind the port. Serving-path SQL fragments derive from the port's attribute vocabulary in exactly one module; equivalence guaranteed by the differential conformance test.

## Constraints

Inherited from the locked intake, Frame, and substrate canon; restated where load-bearing.

- **Substrate:** single-process Postgres + pgvector behind the `Storage` trait (P-0010); A1-clean — no extension beyond pgvector; retrieval additions expressible through the trait, preserving its two co-equal invariants (single-transaction keyed supersession; workspace-scoped isolation). Schema migrations for this cluster's tables inherit the system-wide substrate rule **R-0013-d** (forward-only; work against empty and populated databases; no destructive migration without a verified pre-migration backup) — restated here for cross-reference, not re-locked.
- **Host resource footprint (always-on encoders):** the BGE-M3 + BGE-reranker-v2-m3 artifacts load into the single host process in every deployment; the sizing disclosure (on-disk, resident-memory, load-time orders of magnitude) is carried in P-0014 RA-8 — the first sizing basis for the single-binary posture, measured and refined at implementation.
- **Model-hosting boundary (RC-1):** the system MUST NOT host a *generative* LLM; all generative work calls out to the external model at V0. Local *non-generative* inference (BGE-M3 embed, BGE-reranker rerank) is permitted host-side behind the host-fn seam. The product-brief amendment reconciling every falsified canonical copy rides this cluster's docs change (PD1).
- **Dependency tier:** Green defaults only — `fastembed-rs`, BGE-M3, BGE-reranker-v2-m3 (MIT/Apache-2.0); cloud-LLM use is operational cost under the standing API-key approval. No Yellow/Red dependency without an explicit named trigger.
- **Rust-first; no vendored ports:** external reference implementations (RAPTOR and kin) are design references reimplemented against the substrate.
- **Plugin model:** IO-free cores; inference reached only through host-fns; no plugin receives raw retrieval SQL access; plugins remain leaves (P-0002/P-0012/P-0013).
- **Progressive disclosure is physics:** "return more tokens" is not an acceptable failure mode; every verb budget-enforced (R-0024).
- **Threat model carried:** TB-1..TB-5 controls per the Frame §5, including residuals (i) index-time mislabel egress (maintainer-accepted for V0; P-0015 owns labeling correctness + the polarity revisit), (ii) HyDE query-text egress (config switch is the control), (iii) provider-trust under the standing approval, (iv) indirect prompt injection (controls: R-0030-f for 4.1; R-0032-b for 4.4; R-0030-e untrusted-synthesis marking for 4.3).
- Architecture MUST NOT be schedule-pressured; marketing-tier dates are not architectural inputs.

## Intent self-report

**(1) The intake JTBD as read here:** an agent picking up any artifact gets, in one MCP call, a focused and honest picture of where that artifact sits — what it extends, what it feeds, whether it is current, whether it may be used at all — sized to the agent's context budget, so sessions stop paying a re-derivation tax and stop acting on stale or forbidden context. The envelope is not garnish: a context layer that serves stale or policy-violating context *confidently* is worse than none.

**(2) Spec decisions that strain an enumerated Non-goal or Success criterion:** none diverge; two near-boundary items are inherited from the Frame and held at the same line: (a) R-0034 ships the *signals* while the eval harness that reads them stays a committed-tier deliverable (non-goal 3 — the boundary is deliberate: signals are SC6 scope, the reader is not); (b) R-0035 adds tenancy *structure* on new tables and changes no enforcement posture (non-goal 6 — structural scoping is the intake's own hard constraint). One interpretation carried, not introduced: SC2's singular "source-role" is implemented by the two-axis envelope per the Frame §11 interpretation note (decomposer-ratified; all five candidate-role semantics preserved at V0 via the `admin-only` gate outcome).
