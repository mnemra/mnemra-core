## JTBD

Today a mnemra plugin exists only as files loaded from known local paths, gated by the inner Ed25519 manifest signature (P-0005) and the R-0021 BLAKE3 content-hash over the wasm component. The **distributable unit does not exist as a designed thing**: there is no packaging shape, the package as a whole is unsigned, and non-wasm artifacts (data, assets, schemas, secondary components) are not individually integrity-covered — a concrete per-artifact substitution/tamper hole, live as of the 2026-07-01 V0 signing ceremony.

The job: **a consumer can establish provenance and integrity of the whole bundle before unpacking** — rather than re-deriving trust from the inner manifest alone, after the fact, artifact by artifact. Satisfying it (within the locked frame) means the plugin becomes a **distributable unit**: packaged in one uniform shape, signed end-to-end at the distribution anchor, fetchable from a filesystem or a self-hosted registry with **zero required round-trip to a host we do not control**, verified on fetch *and* re-verified blob-by-blob at load.

## Non-goals

1. **warg as the store.** Ruled out on intrinsic grounds (maintenance-death — ecosystem converged on OCI; unit is a component-package with no home for non-wasm artifacts). Its signed-append-log *concept* may be borrowed later (or obtained via TUF); the store is not adopted.
2. **IPFS / generic CAS / bespoke CAS-zip.** Foreign-ecosystem, no native signing model; bespoke kept only as documented fallback-of-last-resort, not designed here.
3. **Sigstore keyless signing at this tier.** Forward-allowance only — fires on the R-0005-e trip-wire (third-party publishers, or a deployment tolerating an external transparency-log round-trip; self-hosted Fulcio/Rekor as the R-NoExternalHost-compatible heavy path). Keyed-in-tree is this tier's mechanism.
4. **TUF as a built mechanism.** The rollback/rotation/revocation layer is deferred behind the same R-0005-e trip-wire, co-requiring P-0005's Tier-C offline-root custody hardening. What locks now is the verifier *seam* the TUF adapter later slots behind (ratified 2026-07-07 — seam-now, mechanism-later; the P-0010 D5 precedent; "defer the seam too" explicitly rejected). Rollback/freshness protection is an accepted residual until then; the trip-wire is named and fireable, not silent.
5. **SLSA / in-toto provenance attestations.** Deferred to reproducible builds (#1942), whose payoff point they are. The build-time dependency-confusion residual (wkg-pulled deps composed pre-sign) is named for Frame; an interim lockfile-hash pin may be weighed there without pulling #1942 in.
6. **Third-party publisher support.** Non-`core` plugin distribution, key splitting for external publishers, delegated roles — all R-0005-e territory. This bundle designs the single-publisher (`core: true`) distribution story.
7. **Registry product choice / deployment story.** Any OCI-compliant registry satisfies the self-hosted transport (zot noted as aligned); picking and operating one is a deployment decision for a later brief (Q6), not this bundle.
8. **Tier-C custody hardening itself.** P-0005's `{{P-SigningKeyCustodyHardening}}` slot stays unauthored; its trip-wire is unchanged. The paired W2-opt custody-ADR pre-author is **parked** as its own later pickup (ratified 2026-07-07) — this bundle's ADR records the Q5 exposure and trip-wire only. Parked-item cadence (per P-Defer/DF1): W2-opt sits on the strangler-program board, whose per-bundle review is the named human cadence that resurfaces it; the maintainer may pull it earlier.

## Success criteria

1. **Uniform bundle, no bare path, hard cutover.** Every plugin — including wasm-only (N=1) — is an OCI artifact: one manifest binding N≥1 digest-addressed blobs, the component `.wasm` always artifact #1. The invariant lands as **one atomic flip** (ratified 2026-07-07): existing pre-OCI plugins are re-packaged in the same change; no legacy-accept window, no sunset mechanism to design. *Observable:* the loader rejects any plugin not presenting a signed OCI manifest binding every artifact; no code path loads a bare `.wasm`; post-flip, no test exercises a pre-OCI load.
2. **Package signature, verified on fetch.** The OCI manifest digest is signed (domain-separated) and attached as an OCI 1.1 referrer; fetch verifies it under signer-key-pinning before any blob is unpacked or handed on. *Observable:* an unsigned, re-signed-by-unknown-key, or tampered bundle fails closed at fetch.
3. **Both R-NoExternalHost transports work.** Filesystem (OCI image-layout read off disk) and self-hosted server (OCI distribution API on the LAN) are both first-class load sources with no external-host round-trip. *Observable:* fetch + verify + load succeed from a local `oci-layout` directory and from a self-hosted registry in tests; no test requires a public host.
4. **Inner complete-mediation over all artifacts (Decision A).** The inner signed manifest binds every blob via an N≥1 `[[artifacts]]` binding; each blob is verified with the R-0021 single-read / fail-closed discipline at the provenance anchor before its consumer sees it. *Observable:* a substituted non-wasm blob fails at load even when the distribution-layer check is bypassed (e.g., post-fetch cache tamper).
5. **Substitution closed at both anchors.** Per-artifact substitution fails closed at the distribution anchor (digest-pin under the signed OCI manifest) and independently at the provenance anchor (inner gate). *Observable:* substitution tests at each anchor, with the other anchor disabled, both reject.
6. **Contract edit lands verified against source, and the prior deferral is dispositioned.** The `[[artifacts]]` schema, the P-0003 amendment (single `[component]` hash → N≥1 binding), and the plugin-contract edit (task #838 reach) are designed against the *opened* source artifacts — the research brief cites that reach as-given-by-the-maintainer, so Frame must confirm it against P-0003 / P-0019 before the schema locks. P-0019 **DEF-2** (third-party distribution deferral: OCI/wkg + cosign-style verification + `application/vnd.mnemra.plugin.v1` + verify-before-instantiate) already frames this surface; this bundle pulls that deferral deliberately ahead of its third-party tripwire (board placement, single-publisher scope) and supersedes its recorded cosign-on-wkg shape per the locked research — Frame **must disposition DEF-2 explicitly** (amend or record supersession), not leave two live framings. The contract-edit reach also includes: (a) the P-0003 `[[artifacts]]` extension **supersedes the amendment's current "binds bytes, not a path" scope-out** — per-artifact identifiers become necessary for entry↔blob resolution — and every per-artifact identifier **MUST live in the signed canonical body per R-0005-h**, with unambiguous, collision-free entry↔blob resolution (a Frame security constraint, not schema-design discovery); (b) **P-0019 D6's lifecycle sequence** — Frame decides whether D6 gains the distribution pre-phase (fetch → verify-package-signature → verify-blob-digests → unpack, fail-closed, ahead of "discovered") or a new distribution ADR that D6 cross-references; (c) **Q4 as a named lock target**: the `application/vnd.mnemra.plugin.v1` artifactType + per-blob media types lock with the schema, not by assumption. The design must also honor P-0012's Wasmtime pin. *Observable:* the Frame's rationale chain cites the opened artifacts and carries the DEF-2 disposition, the signed-body-residence constraint, the D6-vs-new-ADR decision, and the media-type locks; the finished mechanism gets a security re-verification pass.
7. **Dependency gates pass.** `cargo audit` / RUSTSEC screen against the pinned adopted crates (`oci-client`, `oci-spec`, and `ocidir` if adopted) passes or has each finding dispositioned (L5); `ocidir`'s layout-write path is audited before it becomes the fs primitive, or the narrow read/write is written directly against `oci-spec` (L3). *Observable:* the audit output and disposition are part of the spec's acceptance evidence.
8. **Fail-closed everywhere, with designed errors.** Malformed outer manifest, unknown signer key, missing referrer signature, digest mismatch, media-type mismatch, `[[artifacts]]` binding incomplete, fetch/unpack resource-bound breach (oversized blob, excessive artifact count, decompression bomb) — each rejects the fetch/load with a distinct designed error; no best-effort path. *Observable:* an error-path test per rejection class.

## Hard constraints

1. **H1 — uniform packaging / no bare-load path** (the maintainer, 2026-07-01, LOCKED). One packaging shape, one load path; N=1 content permitted, the bare *path* banned.
2. **R-NoExternalHost** (the maintainer, 2026-07-01, LOCKED). Self-hostability on a filesystem AND a self-hosted internal server; zero required round-trip to an uncontrolled host. Reserve bare "air-gap" for the strict fs-only subset.
3. **Decision A — inner `[[artifacts]]` complete-mediation** (the maintainer, 2026-07-01, LOCKED). The inner signed manifest binds all N≥1 blobs; R-0021's discipline becomes the primary load-time integrity mechanism for every artifact; the distribution-anchor cache re-check is the complementary outer check. Mechanism design (schema, ordering, media types, fail-closed loop) is this bundle's headline work — the decision is not re-litigable, its mechanism is not pre-designed.
4. **Store standard = OCI artifacts** (research-locked). Image-layout for fs, distribution API for server; authored directly (ORAS-style arbitrary-artifact path), not via wkg's one-component-per-package convention; wkg retained only for WIT/component dependency pulls.
5. **Package signing = OCI manifest digest, keyed-in-tree, as OCI 1.1 referrer** (research-locked, refining "cosign by default"). Reuse the P-0005 `ed25519-dalek` root with a domain-separation prefix; verify in-Rust on fetch. Key-custody consequence (Q5, ratified 2026-07-07): package signature and inner manifest signature share one root at this tier — scope-independent but **not compromise-independent**; the split to a distinct distribution key (or delegated roles) fires at R-0005-e. The ADR records this exposure explicitly; it is never silent.
6. **One flat OCI manifest at this tier** (Q2, ratified 2026-07-07). The plugin is an atomic unit: one manifest binding N blobs; the image-index (manifest-of-manifests) shape is noted as the forward path if secondary components ever need independent addressing/signing — noted, not designed.
7. **Existing gates unchanged in primitive.** Inner Ed25519 (P-0005) + BLAKE3 single-read (R-0021) extend in coverage (all blobs), not in kind; P-0005's V0 invariants hold untouched — synchronous fail-closed verification, no verify-async path, `core: true` by signature provenance, file-mode startup checks. The design slots into P-0019 D6's locked lifecycle ordering (signature-verify → content-hash-verify → manifest-parse/allowlist → instantiate, fail-closed at every gate) and targets P-0012's pinned raw-Wasmtime runtime.
8. **Rust-ecosystem, Green-tier dependencies only.** All candidate crates are Green (`Apache-2.0 WITH LLVM-exception` maps Green); license tier is a separate additive gate from stack-fit and from the L5 vulnerability screen.

## Evidence

- **Origin:** the maintainer, 2026-07-01, mid V0 signing ceremony — "when we package the plugin the package itself needs to also be signed" (task #1944).
- **Live gap:** as of 2026-07-01 only the inner manifest (Ed25519) and the wasm component (BLAKE3) are covered; the package is unsigned and non-wasm artifacts are individually uncovered — the substitution/tamper hole this bundle closes (research brief §1).
- **Research:** task #1943 brief, locked 2026-07-01 after three security-review rounds; final revision resolves the load-time-redundancy finding via Decision A. Store/signing/rollback picks stable across all rounds.
- **Use case (structured):** (a) *Restricted-egress LAN install* — an operator points a mnemra host at an internal registry, installs a `core` plugin; fetch verifies the package signature + every blob digest before unpacking; load re-verifies every blob at the provenance anchor. (b) *Air-gapped fs install* — the same bundle read as an `oci-layout` directory from removable media; identical verification chain, no network.
- **Prior decision anchor:** P-0019 DEF-2 (2026-07-02) already deferred exactly this surface — "third-party plugin distribution + signature verification" over an OCI registry with a `application/vnd.mnemra.plugin.v1` artifact type, verify-before-instantiate — recording the survey's cosign-on-wkg shape as *recommended, not locked*. This bundle is that deferral pulled deliberately (maintainer board placement, 2026-07-05) ahead of its third-party tripwire, at single-publisher scope, with the shape refined by the locked #1943 research (keyed-in-tree over cosign; direct OCI over wkg).
- **Ecosystem claims to re-verify at Frame** (research Honesty note): "OCI is the Wasm-component ecosystem default" and "warg is no longer actively developed" are findings verified 2026-07-01, not settled facts.

## Risk profile

**Touches trust boundary: YES** — supply chain, signing, and the plugin load path (build-pipeline ↔ host trust boundaries). Required Stage-2 constraint: the security reviewer's security-mode pass runs at Frame on the known mechanism. Threat anchors carried forward: root-key compromise (Critical — single root signs both layers at this tier, Q5), per-artifact substitution (the gap being closed), fetch↔load TOCTOU (closed by Decision A; verify in mechanism design), rollback/downgrade (accepted residual behind the R-0005-e trip-wire), build-time dependency confusion (named residual, #1942's payoff point), **fetch/unpack resource exhaustion** (new network surface: oversized blob, excessive artifact count N, decompression bomb, unbounded buffering — load-bearing because digest verification must read the bytes first, so availability-DoS lands before the integrity gates; P-0007 covers execution-time limits only, so fetch/unpack-time bounds — max bundle size, max N, streaming-verify, decompression-ratio cap — are a required Frame constraint; LAN transport auth/TLS is defense-in-depth for confidentiality, not the availability answer).

**Load-path invariants carried wholesale:** the research brief §4's fail-closed invariants #1–#10 are **required Frame input in their entirety** — each is locked at Frame/ADR, not re-discovered at implementation. Three named here because the intake's criteria don't otherwise surface them: **#4** — every blob digest is recomputed over the *received* bytes; the registry's `Docker-Content-Digest` header is never trusted; **#6** — the outer manifest's `artifactType` is pinned to the mnemra plugin media type (confused-deputy block, Q4); **#7** — any annotation→filename mapping (e.g. `org.opencontainers.image.title`) passes a path-traversal / zip-slip guard.
